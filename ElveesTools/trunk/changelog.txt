Проэктные файлы соответсвенно:

	ElveesTools71.vcproj
	
[25 апреля 2006]
Для локализации проектов в СThread добавлена функция
- void SetLocale(DWORD dwLCID)
По умолчанию локаль ровна LOCALE_USER_DEFAULT.
инициализировать нужно один раз при старте приложения.

[12 мая 2006]
Исправленна ошибка при запросе Perfdata::Counter.
Можно записывать минидампы при структурных исключениях
Включаются функцией:
- CStructuredException::EnableMiniDump

[30 мая 2006]
Убрал вывод сообщений сокета, тк для некоторых приложений это нормальный режим
работы (пытаться сооединиться с удаленным компьютером). Вывод ошибок теперь
полностью лежит на приложении (в Release)
Исправленна ошибка в UDP сервере (размер пакетов UPD равен максимальному 2^16).

[22 июня 2006]
Кириллица не отображается при выводе в консоль. Сделал форматирование всех сообщений
в локали 0x0409 [English US] если это возможно.

[26 июня 2006]
Исправленна утечка в UDP сервере (не приходили сообщения о посылке пакетов).

[6 июля 2006]
Добавил проектные конфигурации со статически линкуемой CRT.

[20 июля 2006]
Переделана структура декодера. Теперь он может работать через интерфейс ICSendMessage,
если невозможно работать напрямую (цепочки декодеров MPG4->YV12->RGB32).

[17 августа 2006]
Убрал косвенное использование CUsesWinsock. Инициализация Winsock2 теперь должна
делаться в приложении (один раз).
Убрал основной поток UPD сервера, тк чтение можно осуществлять через любой сокет.
посылка событий на чтение нужно делать в приложение до/после получения очередного пакета.

[24 августа 2006]
Убрал "перехват" исключений дебугером (включается вызовом CThread::SetDebugMode)
Ошибка: адресс полученого пакета UDP был привязан к сокету, а не к буферу!

[24 октября 2006]
Добавленны события старта и остановки UDP сервера. Для корректного выхода при синхронизованном
получении дэйтаграмм.
